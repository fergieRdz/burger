# Happy Burger — Descripción del Proyecto

## ¿Qué se hizo?

Se desarrolló un sistema administrativo para una franquicia de hamburguesas llamada **Happy Burger**, utilizando Python como lenguaje principal. El sistema cuenta con dos interfaces: una de consola (CLI) para operación rápida y una interfaz web construida con Flask para gestión visual desde el navegador.

## Objetivo

Aplicar los conceptos de **Programación Orientada a Objetos (POO)** en un caso de uso real, integrando persistencia de datos con SQLite y una interfaz web funcional. El proyecto demuestra el uso de clases, herencia, encapsulamiento y modularización dentro de un sistema administrativo completo.

## ¿Qué puede hacer el sistema?

- Registrar, editar y eliminar **clientes** y **productos del menú**
- Crear **pedidos** con uno o varios productos y generar un ticket `.txt` por cada uno
- Consultar y filtrar pedidos por estado (pendiente / completado) o por fecha
- Ver un **dashboard** con métricas globales: total de clientes, productos, pedidos y ventas
- Visualizar **reportes con gráficas** de productos más vendidos y ventas por día

## Tecnologías usadas

- Python 3
- Flask (interfaz web)
- SQLite (base de datos)
- HTML + CSS + Chart.js (frontend)
